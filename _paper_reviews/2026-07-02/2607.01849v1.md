# Paper Review: Decomposer: Learning to Decompile Symbolic Music to Programs

## 论文类型: 问题定义型 + 方法型（混合）
提出新任务（symbolic music decompilation / MIDI-to-Strudel），并配套两阶段 SFT+RL 方法与合成数据集 Strudel-Synth。问题定义是核心，方法是配套。

## 公理审查结果

### 公理一：对象公理
- 判定: ✅
- 依据: 对象真实可操作：MIDI（低层 note events）→ Strudel（带 pattern 算子、和弦符号、repetition/layering 的 DSL）的逆向映射。论文清楚定义了 faithfulness（执行渲染 MIDI 与输入的多乐器 onset F1）与 readability（12 项 rubric + LLM reranker rank），并把任务嵌入到既有的 decompilation 谱系（binary→source, image→SVG, 3D→CAD）。问题真实存在：heuristic 转换器（beejsbj/midi-strudel、Emanuel-de-Jong/MIDI-To-Strudel）确实只能做 note-by-note 转写，frontier LLM 又无法忠实渲染。可编辑音乐程序对 live coding、音乐教育、motif discovery 是真实下游需求，不是"世界语"型窄场景。
- Wiki 证据: OMC wiki_query 工具在本环境不可用；paper-wiki 对所有相关 concept（symbolic music decompilation、music LLM post-training、Strudel、GRPO、LMD）均返回空。free-search 补充：beejsbj/midi-strudel 和 Emanuel-de-Jong/MIDI-To-Strudel 两个 GitHub 项目证实 heuristic baseline 真实存在且只做逐音转写；openalex 检索 "symbolic music representation learning" 返回 CLaMP 3、music understanding survey、SongComposer，确认 MIDI/符号音乐是活跃研究区。论文自陈 688 个非平凡 human-authored Strudel 程序的爬取量，佐证低资源问题真实。

### 公理二：识别公理
- 判定: ⚠️
- 依据: 论文做了关键变量隔离：SFT alone vs +RL（表 2 同一 backbone）、readability 权重 w2 扫描（图 4）、SFT 初始化 vs 直接 RL（图 5，证明无 SFT 时 RL 崩塌到短 literal note list）。这些是真正的因果识别实验。但缺口：(1) 没有 4B vs 8B 的 compute-matched 比较，无法排除"8B 赢只是因为参数多"；(2) Strudel-Synth 由 Claude-Opus-4.6 蒸馏而来，而 Claude-Opus-4.6 同时是 baseline，存在潜在的知识泄漏，作者未做隔离分析；(3) reward 中 readability judge 用未指明的 LLM（rubric + reranker），judge 身份与训练 reward 是否同源未交代清楚。
- Wiki 证据: paper-wiki 无 baseline 论文记录。free-search 找到 AutoDecompiler (arXiv 2606.16162, RL-trained binary decompilation LLM) 和 Rendering-Aware RL for SVG generation (2505.20793) 两篇同期 execution-based RL decompilation 工作，作者在 Related Work 中已正确引用并定位差异（音乐 + 双 reward + 低资源 DSL）。同期工作（≤2 月），不作为 novelty 扣分，但说明 RL-decompilation 范式已被多人并行采用。

### 公理三：独立性公理
- 判定: ⚠️
- 依据: 训练-评测闭环部分独立，部分循环。faithfulness reward（onset F1 via mir_eval）和评测指标（Onset/Frame/Inst F1）共享同一度量族——但这是物理客观指标，不构成污染。readability 训练 reward（12 项 rubric LLM judge）与评测指标（Rubric score、LLM reranker Rank）共享同一 rubric / 同类 LLM judge，这是 **major** 循环：训练里被奖励的东西正是评测里被测的东西。作者没有引入独立的人类 readability 标注或独立的 judge 模型族作为锚点。合成数据由 Claude-Opus-4.6 生成，评测里 Claude-Opus-4.6 是 baseline，存在轻微数据-评测耦合。结论"Decomposer 比 Claude-Opus-4.6 更忠实"仍然可信（faithfulness 是客观指标），但"更可读"的结论需打折扣。
- Wiki 证据: paper-wiki 无 judge 独立性相关记录。free-search 未找到独立的 Strudel readability 人工标注集。论文自陈 readability rubric 借鉴 Buse & Weimer (2010)、Scalabrino et al. (2018) 的 code-readability metrics，但 LLM-as-judge 的具体模型未在正文披露（仅说 "an LLM judge"），Appendix D 需核查（本 review 未读 appendix 全文）。

### 公理四：压缩公理
- 判定: ✅
- 依据: 方法是 SFT + GDPO（GRPO 变体的多 reward 版本）+ execution-based reward，每个组件都有既定来源但组合是有理由的，不是任意堆叠。关键压缩价值在于：(1) 把"音乐分析-by-synthesis"从参数恢复（differentiable synthesis、physical models）重构为"程序恢复"，结构是 inferred 而非 a priori specified——这是真实的问题重构；(2) 双 reward 解耦（GDPO 分组归一化各 reward 再合并）是针对 faithfulness–readability trade-off 的具体技术回应，图 4 的 w2 扫描实证了 trade-off；(3) SFT warm start 解决 RL 冷启动崩塌（图 5）是一个可迁移的经验结论。命名略有膨胀（"Decomposer" + "Strudel-Synth" + "symbolic music decompilation" 三个新名词），但每个都对应实质对象。
- Wiki 证据: paper-wiki 无记录。free-search 确认 GDPO (Liu et al. 2026)、GRPO (Guo et al. 2025) 是已发表方法，作者正确归因。execution-based RL for decompilation 在 SVG (Rodriguez 2025)、CAD (Kolodiazhnyi 2026) 已有先例，作者在 Related Work 明确承认并说"We extend this paradigm to music"，未夸大为全新范式。

### 公理五：效用公理
- 判定: ✅
- 依据: 表 2 主结果在两个数据集 × 两个 backbone 上全面取胜于 frontier LLM 的 faithfulness，并显著优于 heuristic 的 readability。绝对值：8B + SFT+RL 在 Strudel-Synth 上 Onset F1=0.73、Inst F1=0.72、Rubric=0.74、Comp=0.99，是可用水平；在 LMD 上 Onset=0.60、Inst=0.58、Rubric=0.61，real-world 仍有提升空间但已显著超过 GPT-5.5（Onset 0.27）。表 3 跨 4 个 held-out 数据集（longer LMD、GigaMIDI、NES-MDB、Nottingham）泛化性得到验证，Best@5 Onset F1 全面超 GPT-5.5。诚实讨论了失败模式（longer LMD 与 NES-MDB 上性能更低）。baseline 覆盖：3 个 closed-weight frontier LLM + 1 个 heuristic + 自家 SFT-only / RL-only ablation，公平且充分。8B open-weight 模型击败 closed-weight frontier LLM 在 faithfulness 上是有说服力的效用证据。
- Wiki 证据: paper-wiki 对 LMD 数据集查询返回空。free-search 确认 LMD (Raffel 2016 Lakh MIDI Dataset)、GigaMIDI、NES-MDB (Donahue 2018)、Nottingham 均为真实公开数据集。未找到任何已发表的 MIDI-to-Strudel 同类方法（除作者自己），所以"无直接 prior baseline"成立，作者用 frontier LLM + heuristic 作为 baseline 是合理选择。

### 公理六：新颖性公理
- 判定: ⚠️
- 依据: 真实增量但偏增量。(1) 任务定义"symbolic music decompilation"是真实新任务——MIDI→music DSL 此前无学术工作（仅 GitHub 工程脚本），这是论文最强的贡献。(2) 方法是 A+B+C 组合：SFT（标准）+ GRPO/GDPO（已发表）+ execution-based reward（SVG/CAD decompilation 已有）+ rubric-as-reward（Viswanathan 2025）+ 合成数据蒸馏（Doh 2025）。每个组件单独看都不新，组合到音乐域是新应用。(3) 跨域迁移（binary/image/CAD decompilation → music decompilation）属于公理六允许的"跨领域迁移解决本领域真实问题"，且作者证明了真实问题（heuristic 与 frontier LLM 都不足），所以迁移有效。(4) 缺少新机制解释或反直觉经验规律——SFT warm start 解决 RL 冷启动在 code/math RL 文献里是已知现象，图 5 是再确认而非新发现。综合：任务新颖性高，方法新颖性低。
- Wiki 证据: paper-wiki 对核心 idea 无记录。free-search 找到的同期工作 AutoDecompiler (binary decompilation + RL + multi-turn)、Zero-to-CAD (agentic CAD synthesis without real data)、Rendering-Aware SVG RL 均在 2026 年发表，与本文时间窗重叠，属 concurrent work，不扣分。但它们的存在说明"execution-based RL for decompilation"已是 2025-2026 年的成熟 recipe，本文是这一 recipe 的音乐实例化。

### 公理七：可复现公理
- 判定: ⚠️
- 依据: 论文承诺释放 Strudel-Synth（21,174 pairs，数量明确），项目页 yewon-kim.com/decomposer 存在。训练细节：Qwen3-4B/8B、LoRA SFT 1 epoch、GDPO、reward 权重 (1.0, 0.5)、10K SFT + 10K RL 划分，超参基本充分（Appendix F.4 未读全文）。缺口：(1) readability LLM judge 的具体模型/版本未在正文披露，复现 readability reward 有障碍；(2) Strudel runtime 的 headless 渲染环境是否容器化未说明；(3) 论文发表日 2026-07-02，发布时点尚早，代码/checkpoint 是否已公开需查 project page（本 review 未验证 GitHub 链接）；(4) 依赖 Claude-Opus-4.6 蒸馏合成数据，复现者需付费 API。整体可复现但需中等以上工程投入。
- Wiki 证据: paper-wiki 无记录。free-search 找到 project page 域名存在但未深入抓取内容。

## 总评

- 科学价值: 中 — 把 music analysis-by-synthesis 从参数恢复重构为程序恢复是有认识论意义的框架迁移；但核心方法（execution-based RL decompilation）是已有 recipe 的领域实例化，未产生新的可迁移机制解释。
- 方法价值: 中 — SFT+GDPO+双 reward 的两阶段 pipeline 工程完整、ablation 清晰（图 4、图 5、SFT-only/RL-only 表 2），对音乐 DSL 这一低资源场景的 RL 冷启动有实证贡献；但每个组件单独都不新。
- 社区价值: 高 — 定义了一个真实新任务，释放 21K 合成对数据集，提供 frontier LLM + heuristic 双 baseline，跨 4 个 held-out 数据集的泛化评测，为音乐生成/编辑/理解社区提供了新的可编程接口与评测基础设施。任务定义本身是论文最大的贡献。
